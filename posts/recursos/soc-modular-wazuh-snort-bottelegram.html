SOC casero modular con Wazuh + Snort + Bot de Telegram

Implementación técnica de un SOC casero usando Wazuh, Snort y un bot de Telegram para monitorización, detección y envío de alertas de seguridad en tiempo real.

Introducción

Este post documenta la implementación técnica de un sistema de monitorización, detección y notificación de alertas de seguridad usando herramientas open source accesibles. El stack completo está basado en Wazuh 4.9.2, Snort 2.9.x en Windows y un bot de Telegram para enviar notificaciones del nivel de criticidad que configures en tiempo real.

La arquitectura resultante es la siguiente:

⚠️ Este contenido es de carácter educativo. Todo fue implementado en un entorno controlado y virtualizado.

1. Wazuh — Instalación y configuración base

1.1 Despliegue de la VM

Wazuh ofrece una máquina virtual preconfigurada en formato .ova que incluye:

  • Wazuh Manager
  • Wazuh Dashboard
  • Wazuh Indexer
  • Filebeat-OSS
  • Amazon Linux

Descarga desde la documentación oficial de Wazuh.
Importa el .ova en VirtualBox desde Archivo → Importar servicio virtualizado.

Requisitos mínimos recomendados:

RecursoMínimo
CPU2 núcleos
RAM2-4 GB
Almacenamiento20 GB
RedNAT / Adaptador puente

1.2 Acceso al Dashboard

Una vez arrancada la VM, obtén su IP:

ip a

Accede desde el navegador del host: https://192.168.1.X
Credenciales por defecto: admin / admin. Cámbialas desde el Dashboard → icono de usuario → Reset password.

1.3 Instalación del agente en Windows

Desde la terminal del servidor Wazuh como root:

/var/ossec/bin/manage_agents

Selecciona A para añadir agente, introduce nombre e IP del cliente Windows. Extrae la clave de autenticación con opción E.

En el cliente Windows instala el agente desde la guía oficial, introduce la IP del servidor y la clave.
Verifica en: Dashboard → Server management → Endpoints Summary

2. Snort — Instalación y configuración en Windows

2.1 Instalación

Descarga Snort desde snort.org/downloads. Sigue el asistente con valores por defecto. Durante la instalación se pedirá instalar Npcap para captura de paquetes — descárgalo desde npcap.com.

2.2 Reglas preconfiguradas para Windows

Para evitar adaptar manualmente rutas y variables, usa las reglas preconfiguradas del repositorio del TFG:

👉 github.com/Jmullorru/Snort-Rules-Config_Windows

Descarga el ZIP y extrae en c:\Snort confirmando el reemplazo de archivos. Estructura resultante:

c:\Snort\
├── bin\      ├── etc\      ├── rules\
├── preproc_rules\          └── so_rules\

2.3 Personalización de variables de red

Edita c:\snort\etc\snort.conf y ajusta HOME_NET:

# Obtén tu IP
ipconfig
# Setup the network addresses you are protecting
ipvar HOME_NET 192.168.1.X

2.4 Validación de la configuración

Abre CMD como administrador:

cd \snort\bin
snort -W                                    # listar interfaces
snort -c c:\snort\etc\snort.conf -T -i #   # # = número de interfaz

Resultado esperado: Snort successfully validated the configuration!

3. Integración Snort → Wazuh

3.1 Carpeta compartida en VirtualBox

Con la VM de Wazuh apagada ve a Configuración → Carpetas compartidas:

  • Ruta: c:\snort\log
  • Nombre: log
  • Automontar: ✅   Hacer permanente: ✅

Al arrancar, la carpeta estará disponible en /media/sf_log.

3.2 Configuración de Filebeat

vim /etc/filebeat/filebeat.yml

Añade justo después de filebeat.modules:

filebeat.inputs:
  - type: log
    enabled: true
    paths:
      - /media/sf_log/alert.ids
    fields:
      log_type: snort
    fields_under_root: true
filebeat test config    # → Config OK
filebeat test output    # → Conexión OK con Elasticsearch
filebeat -e -d "input"  # → Verificar monitorización en tiempo real

3.3 Validación con wazuh-logtest

/var/ossec/bin/wazuh-logtest

Inserta un log de ejemplo:

12/21-23:12:18.777511 [**] [1:1000002:1] Alerta Critica - Intento de acceso SSH
detectado [**] [Priority: 1] {TCP} 192.168.1.XX:63855 -> 192.168.1.XX:22

Wazuh lo procesará en 3 fases: Pre-decoding → Decoding (IPs, protocolo, fuente: snort) → Filtering (asigna regla y nivel de alerta).

3.4 Reglas y decoders personalizados

Decoder (/var/ossec/etc/decoders/local_decoder.xml):

<decoder name="snort">
  <type>ids</type>
  <prematch>[**] [\d+:\d+:\d+] </prematch>
</decoder>

<decoder name="snort_ssh">
  <parent>snort</parent>
  <type>ids</type>
  <prematch>[**] [\d+:\d+:\d+] </prematch>
  <order>snortRule, alertType, alertInfo, priority, priorityLvl, protocol, srcIP, dstIP, port</order>
</decoder>

Reglas (/var/ossec/etc/rules/local_rules.xml):

<group name="snort_ssh">
  <rule id="100010" level="12">
    <decoded_as>snort_ssh</decoded_as>
    <description>Snort: Alerta Crítica - Intento de acceso SSH detectado</description>
  </rule>
</group>

<group name="snort_portScan">
  <rule id="100011" level="12">
    <decoded_as>snort_portScan</decoded_as>
    <description>Snort: Escaneo de puertos detectado</description>
  </rule>
</group>

3.5 Monitorización de rutas en tiempo real

En C:\Program Files (x86)\ossec-agent\ossec.conf, después del bloque de 32 bits:

<directories realtime="yes" report_changes="yes" check_all="yes">
  C:\Users\TU_USUARIO\Desktop
</directories>
⚠️ Nota Wazuh 4.9.2: Las alertas de Snort se procesan correctamente pero pueden no visualizarse en el Dashboard. Limitación conocida en la comunidad, probablemente ya resuelta en versiones posteriores. Las alertas sí funcionan a nivel de reglas y notificaciones Telegram.

4. Bot de Telegram para alertas en tiempo real

Último paso: un bot de Telegram que reciba las alertas de Wazuh y las reenvíe al instante. Configura bien el nivel mínimo de alerta que te enviará si n oquieres ir a Arkham porque tu teléfono no parará de sonar cual tlf de un físico que ha descubierto como teletransportar materia de un punto A a un punto B.

4.1 Creación del bot con BotFather

  1. Abre Telegram → busca @BotFather (perfil verificado con ✓)
  2. Ejecuta /newbot
  3. Asigna nombre y username terminado en _bot
  4. Guarda el token de API
  5. Ejecuta /start en el bot creado

4.2 Creación del grupo de notificaciones

Crea un grupo en Telegram (ejemplo: MiSOC_WazuhAlerts) y añade el bot como miembro.
El Chat ID lo encuentras en la URL de Telegram Web:

web.telegram.org/a/#-4793XXXX
                    ^^^^^^^^^^^ este es el ID (incluye el guión -)

4.3 Integración en Wazuh

Crea el script wrapper:

nano /var/ossec/integrations/custom-telegram
#!/bin/sh
WPYTHON_BIN="framework/python/bin/python3"
SCRIPT_PATH_NAME="$0"
DIR_NAME="$(cd $(dirname ${SCRIPT_PATH_NAME}); pwd -P)"
SCRIPT_NAME="$(basename ${SCRIPT_PATH_NAME})"
case ${DIR_NAME} in
  */active-response/bin | */wodles*)
    WAZUH_PATH="$(cd ${DIR_NAME}/../..; pwd)"
    PYTHON_SCRIPT="${DIR_NAME}/${SCRIPT_NAME}.py" ;;
  */bin)
    WAZUH_PATH="$(cd ${DIR_NAME}/..; pwd)"
    PYTHON_SCRIPT="${WAZUH_PATH}/framework/scripts/${SCRIPT_NAME}.py" ;;
  */integrations)
    WAZUH_PATH="$(cd ${DIR_NAME}/..; pwd)"
    PYTHON_SCRIPT="${DIR_NAME}/${SCRIPT_NAME}.py" ;;
esac
${WAZUH_PATH}/${WPYTHON_BIN} ${PYTHON_SCRIPT} "$@"

Crea el script Python:

nano /var/ossec/integrations/custom-telegram.py
#!/usr/bin/env python
import sys, json, requests

CHAT_ID="-TU_CHAT_ID"  # sustituir por el ID del grupo

alert_file = open(sys.argv[1])
hook_url   = sys.argv[3]
alert_json = json.loads(alert_file.read())
alert_file.close()

alert_level = alert_json['rule']['level']       if 'level'       in alert_json['rule']  else "N/A"
description = alert_json['rule']['description'] if 'description' in alert_json['rule']  else "N/A"
agent       = alert_json['agent']['name']        if 'name'        in alert_json['agent'] else "N/A"

msg_data = {'chat_id': CHAT_ID, 'text': {
    'description': description,
    'alert_level': str(alert_level),
    'agent': agent
}}
headers = {'content-type': 'application/json', 'Accept-Charset': 'UTF-8'}
requests.post(hook_url, headers=headers, data=json.dumps(msg_data))
sys.exit(0)

Asigna permisos (ajuste necesario en Wazuh 4.9.2):

chown root:wazuh /var/ossec/integrations/custom-telegram*
chmod 750 /var/ossec/integrations/custom-telegram*

4.4 Configuración en ossec.conf

Edita /var/ossec/etc/ossec.conf — sustituye *API_KEY* por el token de BotFather:

<integration>
  <name>custom-telegram</name>
  <level>3</level>
  <hook_url>https://api.telegram.org/bot*API_KEY*/sendMessage</hook_url>
  <alert_format>json</alert_format>
</integration>
systemctl restart wazuh-manager

Conclusión

El stack Wazuh + Snort + Telegram permite construir un sistema de monitorización funcional con herramientas completamente gratuitas y open source. La integración entre las tres herramientas es viable aunque la visualización de alertas de Snort en el Dashboard de Wazuh 4.9.2 presenta limitaciones conocidas — probablemente ya resueltas en versiones posteriores.

Las líneas de mejora naturales son la incorporación de un SOAR (TheHive + Cortex) para automatizar respuestas, el bloqueo automático de IPs y la escalada a entornos con mayor número de agentes.

Conclusion

The Wazuh + Snort + Telegram stack lets you build a functional monitoring system with entirely free, open source tools. Integration between the three tools works well, though Snort alert visualization in the Wazuh 4.9.2 Dashboard has known limitations — likely resolved in later versions.

Natural improvement paths include adding a SOAR (TheHive + Cortex) to automate responses, automatic IP blocking, and scaling to environments with more agents.
